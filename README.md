Week 1: Building Foundations in Problem Solving
In the first week, I focused on getting comfortable with competitive programming and problem-solving in Python. I solved introductory problems involving loops, arrays, sorting, searching, and greedy techniques. Problems such as Weird Algorithm, Missing Number, Number Spiral, Apartments, Ferris Wheel, and Concert Tickets helped me understand how to think systematically about a problem before writing code.
One of the biggest challenges was translating a problem statement into an efficient algorithm. As a beginner, I often knew what needed to be done conceptually but struggled to implement it correctly in Python. This week helped me improve my debugging skills and gain confidence in using built-in Python data structures and sorting techniques

Week 2: Introduction to Recursion, Backtracking, and Dynamic Programming
During the second week, I explored more advanced problem-solving approaches. Problems like Apple Division and Chessboard and Queens introduced me to recursion and backtracking, where I learned how to systematically explore multiple possibilities and prune invalid choices.
I also encountered Dynamic Programming for the first time through problems such as Dice Combinations and Minimizing Coins. Initially, understanding how to store and reuse previously computed results was challenging. However, after solving several DP problems, I started recognizing patterns and appreciating how DP can significantly reduce computation time.
This week taught me the importance of breaking complex problems into smaller subproblems and thinking about optimal solutions rather than brute-force approaches

Week 3: Strengthening Dynamic Programming and Algorithmic Thinking
In the third week, I worked on more challenging Dynamic Programming and optimization problems such as Book Shop, Edit Distance, Array Description, and Counting Towers. I also practiced greedy and sorting-based problems like Restaurant Customers and Movie Festival.
The major challenge this week was understanding state definitions and transitions in Dynamic Programming. Problems like Edit Distance required careful thinking about how previous states influence the current answer. Through repeated practice, I became more comfortable identifying DP states and writing efficient solutions.
This week significantly improved my algorithmic thinking and showed me how different techniques such as sorting, greedy algorithms, recursion, and dynamic programming can be combined to solve complex problems

Overall Learning Experience
As a Python beginner learning Data Structures and Algorithms, this journey helped me move beyond simply writing code and start thinking like a problem solver. I learned how to analyze constraints, choose appropriate algorithms, optimize solutions, and debug effectively. While many problems were initially intimidating, solving them step by step improved my confidence and strengthened my understanding of core DSA concepts. The experience also highlighted areas where I need further practice, especially Dynamic Programming and advanced data structures, which I plan to continue exploring in the future
