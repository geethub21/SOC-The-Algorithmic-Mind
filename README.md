Week 1: Building Foundations in Problem Solving
In the first week, I focused on getting comfortable with competitive programming and problem-solving in Python. I solved introductory problems involving loops, arrays, sorting, searching, and greedy techniques. Problems such as Weird Algorithm, Missing Number, Number Spiral, Apartments, Ferris Wheel, and Concert Tickets helped me understand how to think systematically about a problem before writing code.
One of the biggest challenges was translating a problem statement into an efficient algorithm. As a beginner, I often knew what needed to be done conceptually but struggled to implement it correctly in Python. This week helped me improve my debugging skills and gain confidence in using built-in Python data structures and sorting techniques

Week 2: Introduction to Recursion, Backtracking, and Dynamic Programming
During the second week, I explored more advanced problem-solving approaches. Problems like Apple Division and Chessboard and Queens introduced me to recursion and backtracking, where I learned how to systematically explore multiple possibilities and prune invalid choices.
I also encountered Dynamic Programming for the first time through problems such as Dice Combinations and Minimizing Coins. Initially, understanding how to store and reuse previously computed results was challenging. However, after solving several DP problems, I started recognizing patterns and appreciating how DP can significantly reduce computation time.
This week taught me the importance of breaking complex problems into smaller subproblems and thinking about optimal solutions rather than brute-force approaches

Week 3: Strengthening Dynamic Programming and Algorithmic Thinking
In the third week, I worked on more challenging Dynamic Programming and optimization problems such as Book Shop, Edit Distance, Array Description, and Counting Towers. I also practiced greedy and sorting-based problems like Restaurant Customers and Movie Festival.
The major challenge this week was understanding state definitions and transitions in Dynamic Programming. Problems like Edit Distance required careful thinking about how previous states influence the current answer. Through repeated practice, I became more comfortable identifying DP states and writing efficient solutions.
This week significantly improved my algorithmic thinking and showed me how different techniques such as sorting, greedy algorithms, recursion, and dynamic programming can be combined to solve complex problems

Overall Learning Experience
As a Python beginner learning Data Structures and Algorithms, this journey helped me move beyond simply writing code and start thinking like a problem solver. I learned how to analyze constraints, choose appropriate algorithms, optimize solutions, and debug effectively. While many problems were initially intimidating, solving them step by step improved my confidence and strengthened my understanding of core DSA concepts. The experience also highlighted areas where I need further practice, especially Dynamic Programming and advanced data structures, which I plan to continue exploring in the future

Week 4: During the fourth week, I transitioned from Dynamic Programming to Graphs and Trees, which introduced me to algorithms for solving connectivity, traversal, and shortest path problems. I worked on problems such as Counting Rooms, Building Roads, Message Route, Shortest Routes I, and Tree Diameter, which strengthened my understanding of graph representations using adjacency lists and fundamental traversal techniques like Depth-First Search (DFS) and Breadth-First Search (BFS). I also learned how connected components can be identified efficiently and how graph traversal can be used to determine the shortest path in unweighted graphs. The Shortest Routes I problem introduced me to Dijkstra's Algorithm, where I gained experience using priority queues to compute minimum distances in weighted graphs. Solving Tree Diameter helped me understand tree properties and the technique of performing multiple traversals to determine the longest path in a tree. Initially, visualizing graph structures and selecting the appropriate traversal algorithm for different problem types was challenging. However, by practicing a variety of graph problems, I became more confident in modeling real-world scenarios as graphs, implementing efficient traversal algorithms, and analyzing their time complexity. This week significantly improved my understanding of graph theory and provided a strong foundation for tackling more advanced graph and tree algorithms in future week

Week 5: During the fifth week, I continued building my understanding of graph algorithms by solving problems such as Tree Distances II, Building Teams, Road Reparation, Course Schedule, Longest Flight Route, and Round Trip II. These problems introduced me to important concepts including bipartite graphs, topological sorting, minimum spanning trees, longest paths in Directed Acyclic Graphs (DAGs), and cycle detection. I learned how different graph problems require different algorithmic approaches, such as Kruskal's Algorithm with Disjoint Set Union (DSU) for minimum spanning trees and DFS-based techniques for traversal and cycle detection. Initially, identifying the most suitable algorithm for each problem was challenging, but with regular practice I became more confident in recognizing graph patterns and implementing efficient solutions. This week strengthened my understanding of advanced graph algorithms and improved my ability to solve complex graph-based problem

Week 6: During the sixth week, I explored range queries, mathematical algorithms, and introductory game theory through problems such as Static Range Sum Queries, Static Range Minimum Queries, Dynamic Range Sum Queries, Counting Divisors, Exponentiation, and Nim Game. These problems helped me understand efficient techniques for answering range queries using prefix sums and segment trees, along with mathematical concepts like fast modular exponentiation and divisor counting. I was also introduced to the basic principles of game theory through the Nim Game. Initially, implementing efficient data structures for range queries was challenging, but with practice I became more comfortable optimizing solutions and selecting the appropriate algorithm for different problem types. This week improved my understanding of mathematical problem-solving and efficient query processing in competitive programming

Week 7: During the seventh week, I explored advanced topics including string algorithms, combinatorics, and number theory by solving problems such as String Matching, Word Combinations, Required Substring, Bracket Sequences I, and Counting Necklaces. These problems introduced me to efficient string processing techniques, pattern matching, and combinatorial counting methods. I also gained a better understanding of dynamic programming on strings and the application of modular arithmetic in counting problems. Initially, implementing string-based algorithms and solving combinatorial problems efficiently was challenging, but with consistent practice I became more confident in identifying suitable approaches and writing optimized solutions. This week expanded my knowledge of advanced algorithmic techniques and strengthened my ability to solve complex problems involving strings and mathematical reasoning
